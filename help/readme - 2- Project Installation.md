# *************************************************************** HR System – راهنمای راه‌اندازی (مرحله ۲)
## آماده‌سازی پروژه و نصب وابستگی‌ها

در این مرحله فرض می‌کنیم که:
- فایل `HR.zip` را دریافت کرده‌اید
- هنوز هیچ کاری روی پروژه انجام نداده‌اید
- اولین بار است که می‌خواهید پروژه را اجرا کنید

در پایان این مرحله:
- پروژه روی سیستم شما کپی شده است
- Virtual Environment ساخته شده است
- تمام کتابخانه‌های Python نصب شده‌اند
- پروژه آماده اجرا است (ولی هنوز اجرا نشده)

---

## 1️⃣ دریافت و استخراج پروژه

### مرحله 1: کپی فایل پروژه

1. فایل `HR.zip` را دریافت کنید
2. یک مسیر ساده و بدون فاصله انتخاب کنید، مثلاً:
C:\Projects\


3. فایل `HR.zip` را در این مسیر کپی کنید
4. روی فایل راست‌کلیک کرده و **Extract Here** یا **Extract All** را انتخاب کنید

بعد از Extract، باید ساختاری شبیه این ببینید:

C:\Projects\HR
backend
frontend
logs
README.md


> ⚠️ نکته مهم  
> از قرار دادن پروژه در مسیرهایی مثل `Desktop` یا `Downloads` خودداری کنید.

---

## 2️⃣ باز کردن Command Prompt

1. دکمه **Start** ویندوز را بزنید
2. عبارت `cmd` را تایپ کنید
3. روی **Command Prompt** کلیک کنید

سپس به مسیر backend بروید:

```bat
cd C:\Projects\HR\backend


## 3️⃣ ساخت Virtual Environment (venv)

چون پوشه venv داخل فایل ZIP نیست، باید آن را خودتان بسازید.

در همان مسیر backend دستور زیر را اجرا کنید:

python -m venv venv

اگر دستور بدون خطا اجرا شد:

یک پوشه جدید به نام venv ساخته می‌شود

ساختار باید به این شکل شود:

backend\
    venv\
    app\
    requirements.txt
    run.py

## 4️⃣ فعال‌سازی Virtual Environment

برای فعال‌سازی venv دستور زیر را بزنید:

venv\Scripts\activate

اگر موفق باشد:

ابتدای خط فرمان چیزی شبیه این می‌بینید:

(venv) C:\Projects\HR\backend>

## 5️⃣ نصب کتابخانه‌های پروژه

در حالی که venv فعال است، دستور زیر را اجرا کنید:

pip install -r requirements.txt

این دستور:

تمام کتابخانه‌های مورد نیاز پروژه را نصب می‌کند

ممکن است چند دقیقه طول بکشد

اگر نصب با موفقیت انجام شود:

خطایی نمایش داده نمی‌شود

پیام‌های نصب پشت سر هم نمایش داده می‌شوند

## 6️⃣ اگر اینترنت مشکل داشت چه کار کنیم؟

اگر هنگام نصب با خطای اینترنت مواجه شدید:

بررسی کنید که از mirror داخلی استفاده می‌کنید

یا کتابخانه‌ها به‌صورت آفلاین در اختیار شما قرار داده شده‌اند

در این صورت ممکن است دستور نصب به این شکل باشد:

pip install -r requirements.txt -i https://pypi.ir/simple

یا (در حالت آفلاین):

pip install --no-index --find-links=packages -r requirements.txt

## 7️⃣ بررسی نهایی این مرحله

در پایان این مرحله، باید شرایط زیر برقرار باشد:

پوشه venv وجود دارد

(venv) در ابتدای Command Prompt دیده می‌شود

دستور pip list اجرا می‌شود و پکیج‌ها را نشان می‌دهد

هیچ خطای قرمز یا Error در Command Prompt وجود ندارد
